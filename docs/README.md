# Documentación del Proyecto

Este directorio contiene toda la documentación del proyecto de automatización de pruebas.

## Archivos principales

### [test-plan.md](test-plan.md)
Plan completo de pruebas del proyecto con:
- Objetivos y alcance
- Estrategia de pruebas
- Criterios de aceptación
- Cronograma
- Recursos necesarios
- Riesgos y mitigación

### [bug-reports.md](bug-reports.md)
Registro de bugs encontrados durante las pruebas con:
- Plantilla de reporte de bugs
- Bugs reportados
- Estadísticas de bugs

### [ai-analysis.md](ai-analysis.md)
Análisis y recomendaciones basadas en IA con:
- Análisis de cobertura
- Optimizaciones sugeridas
- Patrones identificados
- Mejoras futuras

## Estructura general de la documentación

```
docs/
├── test-plan.md      # Plan de pruebas
├── bug-reports.md    # Reporte de bugs
└── ai-analysis.md    # Análisis con IA
```

## Cómo usar esta documentación

1. **Para comenzar**: Lee [test-plan.md](test-plan.md)
2. **Para reportar un bug**: Usa la plantilla en [bug-reports.md](bug-reports.md)
3. **Para optimizaciones**: Consulta [ai-analysis.md](ai-analysis.md)
